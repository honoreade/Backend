This demo shows two different approaches to implementing CRUD (Create, Read, Update, Delete) operations in Python Flask:

Traditional Approach

This implementation uses the traditional Flask approach with page reloads. Each action (create, read, update, delete) causes the page to reload.

Technologies: Python, Flask, Jinja2, SQLAlchemy

View Demo

Modern Approach

This implementation uses JavaScript and AJAX to perform CRUD operations without page reloads, providing a smoother user experience.

Technologies: Python, Flask, JavaScript, Fetch API

View Demo

Technical Details

Both implementations use the same backend API and SQLite database. The difference is in how the frontend interacts with the backend:

  • Traditional: Form submissions and page reloads
  • Modern: JavaScript fetch API and DOM manipulation

The API endpoints (/api/users) handle all CRUD operations and return JSON responses.
